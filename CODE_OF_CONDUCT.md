# Code of Conduct

## Our standard

Gnosys is a small open-source project. We aim for a respectful, productive
community where contributors can collaborate without friction.

We expect everyone interacting with this project — in issues, pull requests,
discussions, and any related space — to:

- **Be respectful** of other contributors, regardless of background or
  experience level.
- **Stay on topic.** Comments should focus on the code, the project, or
  the discussion at hand.
- **Assume good faith.** Most disagreements are technical, not personal.
- **Accept constructive feedback** gracefully and offer it the same way.
- **Help newcomers.** If a question seems basic, it's an opportunity to
  document something better.

## What's not OK

- Personal attacks or insults directed at other contributors.
- Sustained off-topic disruption that prevents normal project work.
- Deliberate trolling or bad-faith argument.
- Sharing private correspondence without consent.
- Threats of any kind.

## Enforcement

Maintainers may, at their discretion, edit or remove comments and
contributions that don't align with this code, and may temporarily or
permanently restrict participation by anyone whose behavior repeatedly
disrupts the project.

## Reporting

If you believe someone is acting in a way that violates this code, please
contact the maintainers privately at the email listed in `package.json`
(`author.url` → see contact info there). Reports are reviewed in good
faith and kept confidential where possible.

## Scope

This code applies to all project spaces — the GitHub repository, issue
tracker, pull requests, releases, and any other channels operated under
the Gnosys name. It also applies to public spaces when an individual is
representing the project.

---

By contributing to Gnosys, you agree to abide by these terms.
